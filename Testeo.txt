## 1.  testear punto por punto

1. **Armar una tabla/lista** (en Excel, Notion, Google Sheets, lo que uses) con columnas tipo:

   * ID / Sprint
   * Sección / Pantalla
   * Rol (Paciente / Especialista / Admin)
   * Requisito (copiado/resumido de la consigna)
   * ¿Cumple? (Sí / No)
   * Comentarios / bugs

2. **Recorrer la app por rol**, no por pantalla:

   * Crear usuario Paciente de prueba
   * Crear Especialista de prueba
   * Crear Admin de prueba
   * Con cada uno, recorrer todas las secciones que le corresponden y completar el checklist.

3. Para cada ítem:

   * Hacer **al menos**:

     * 1 caso “feliz” (todo bien).
     * 1 caso de error (datos inválidos, estados que no permiten la acción, etc.).

4. Al final, revisás la tabla: todo lo que quede en “NO” o con comentarios raros, son cosas a corregir antes de entregar.

## 3. Ejemplo rápido de caso de prueba

Para que no sea solo checklist, te muestro cómo podés documentar un test (muy resumido):

**Caso:** Registro de Paciente exitoso

* **Rol:** Visitante (no logueado)
* **Precondiciones:** No existe usuario con el mail `paciente@test.com`.
* **Pasos:**

  1. Ir a página de Bienvenida → clic en “Registrarse”.
  2. Elegir tipo “Paciente”.
  3. Completar todos los campos válidos.
  4. Subir 2 imágenes.
  5. Resolver el captcha.
  6. Enviar formulario.
* **Resultado esperado:**

  * Se muestra mensaje de registro exitoso.
  * Se envía mail de verificación (si lo implementaste).
  * El usuario aparece en la base de datos como Paciente y no puede loguearse hasta verificar el mail.
